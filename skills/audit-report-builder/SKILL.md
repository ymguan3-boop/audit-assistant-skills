# 調查報告彙整技能

依審計機關聯合稽察調查報告格式，從調查計畫與工作底稿自動產出彙整報告。

## 格式規格摘要

| 層級 | 左縮排 | 首行縮排 | 粗體 |
|------|--------|----------|------|
| **L1**（壹、貳…） | 0 | 0 | ✓ |
| **L2**（一、二…） | 1字元(320twips) | -2字元(-640twips) | ✓ |
| **L3**（(一)(二)） | 3字元(960twips) | -2字元(-640twips) | ✓ |
| **L4**（1.2.3.） | 6字元(1920twips) | -2字元(-640twips) | ✓ |
| **Body** | 4字元(1280twips) | 2字元(640twips) | ✗ |
| **Table title** | 14pt 置中 | — | ✗ |

- 字型：標楷體
- 字型大小：16pt（表格標題14pt，表身12pt，來源/單位註記10pt）
- 固定行高：28pt
- 全形括號：半形 `(` `)` 自動轉 `（` `）`

## 觸發方式

使用者說「幫我彙整調查報告」「做報告」「audit-report」時載入本技能。

## 工作流程

### Step 1：收集使用者提供的檔案

向使用者詢問以下資訊：

> 請提供：
> 1. **調查計畫**檔案路徑（.doc 或 .docx）
> 2. **工作底稿**檔案路徑（可多份，.doc 或 .docx）

若使用者上傳檔案（在對話中附加），先將檔案存至暫存目錄 `C:\Users\ymguan\AppData\Local\Temp\opencode\` 再取得路徑。

若使用者直接提供路徑，驗證檔案存在。

### Step 2：執行彙整

```powershell
python "C:\Users\ymguan\.config\opencode\skills\audit-report-builder\build_report.py" `
  --plan "<調查計畫路徑>" `
  --workpapers "<工作底稿(1)路徑>" `
  --workpapers "<工作底稿(2)路徑>" `
  --output "<輸出檔案路徑>"
```

### Step 3：回傳結果

- 告知使用者產出檔案路徑
- 可開啟 Word 比對格式是否正確

## 注意事項

1. 調查計畫支援 .doc 及 .docx 格式（.doc 需安裝 pywin32）
2. 工作底稿支援 .doc 及 .docx 格式
3. 樣板檔使用內建 template/調查報告.docx
4. 預設輸出至執行目錄下「調查報告_彙整完成.docx」
5. 未安裝 pywin32 時只能讀取 .docx 檔案
6. 報告章節結構：壹(查核緣起)→貳(查核依據)→參(查核範圍)→肆(查核限制)→伍(查核重點)→陸(查核事實)→柒(查核意見)→捌(擬議處理意見)→玖(附錄)
