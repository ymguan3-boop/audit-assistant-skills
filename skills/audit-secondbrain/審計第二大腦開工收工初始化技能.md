# OpenCode 懶人包 #07：開工/收工/初始化技能

> 版本：v0.1
> 更新日期：2026-05-19

---

## 這個懶人包會幫你做什麼？

安裝 OpenCode 全域技能（SKILL.md），讓你的開發工作流程自動化：

- **startup（開工）**：說「開工」→ 自動讀 Obsidian 工作筆記 + 檢查 Git 狀態
- **shutdown（收工）**：說「收工」→ 自動 Git commit/push + 更新 Obsidian 紀錄 + chezmoi 同步
- **project-init（專案初始化）**：說「初始化專案」→ 自動建 AGENTS.md + Git + GitHub + Obsidian

---

## 先備條件

- [ ] 已完成 **懶人包 #03：建立第二大腦 Obsidian**
- [ ] GitHub CLI 已登入

---

## 請 OpenCode 幫我執行以下步驟

### 步驟一：建立 skills 目錄

```bash
mkdir -p ~/.config/opencode/skills/startup
mkdir -p ~/.config/opencode/skills/shutdown
mkdir -p ~/.config/opencode/skills/project-init
```

### 步驟二：建立 startup 技能

建立 `~/.config/opencode/skills/startup/SKILL.md`：

---
name: startup
description: 開工自動同步 — 說「開工」「我來了」「上次做到哪」時載入
---

當使用者說「開工」時，依序執行：
1. 確認工作目錄是否在 GDrive git repo
2. 讀 Obsidian 工作筆記的「上次做到哪」與「下一步」
3. 檢查本地 git status
4. 檢查遠端有無新 commit
5. 給結構化摘要與建議下一步

**原則**：開工只讀不寫。

### 步驟三：建立 shutdown 技能

建立 `~/.config/opencode/skills/shutdown/SKILL.md`：

---
name: shutdown
description: 收工自動同步 — 說「收工」「下班」「結束」時載入
---

當使用者說「收工」時，依序執行：
1. 專案 Git 同步（commit + push，等使用者確認 commit 訊息）
2. chezmoi 同步（`~/.config/opencode/` 或共用 dotfiles 設定變動）
3. 敏感檔案提醒（API key 等）
4. Obsidian 更新（補「最近更動紀錄」）
5. 最終 checklist 回報

### 步驟四：建立 project-init 技能

建立 `~/.config/opencode/skills/project-init/SKILL.md`：

---
name: project-init
description: 新專案初始化 — 說「初始化專案」時載入
---

當使用者說「初始化專案」時，依序執行：
1. 建立 AGENTS.md（含專案名稱、簡介、資料夾結構）
2. Git init + .gitignore + initial commit
3. GitHub 私有 repo 建立 + push
4. Obsidian 對應資料夾與工作流程筆記建立
5. 更新 AGENTS.md 同步對照表
6. 完成報告

### 步驟五：寫入 opencode.json 權限設定

確認 `~/.config/opencode/opencode.json` 中有：

```json
{
  "permission": {
    "skill": {
      "startup": "allow",
      "shutdown": "allow",
      "project-init": "allow",
      "*": "ask"
    }
  }
}
```

### 步驟六：驗證技能

重新開啟 OpenCode，然後說「開工」，確認技能被載入。

## 常見問題

| 問題 | 解法 |
|------|------|
| skill 沒出現在可用清單 | 確認 SKILL.md 檔名全大寫、frontmatter 格式正確 |
| 「開工」無反應 | 重新開啟 OpenCode 讓它掃描技能目錄 |
