# 審計第二大腦設定指南

> 完整詳細步驟請見同目錄下的 `審計第二大腦設定指南Obsidian_(安裝及調查計畫與工作底稿生成).md`

## 什麼是審計第二大腦？

一個基於 Obsidian + Claude Code + MCP Vault 的審計專屬 AI 知識管理系統，能自動：

- ✅ 批次轉換 docx/xlsx/pdf/圖片 → markdown
- ✅ 自動分類查核報告、法規、查核意見
- ✅ 生成調查計畫（含 MD + WORD）
- ✅ 生成相關法規及連結（自動查詢 PCode）
- ✅ 生成調閱資料清單
- ✅ 生成缺失評估問卷
- ✅ 生成標準化工作底稿（四階段結構）
- ✅ 每週自動知識重整

## 系統架構

```
資料蒐集/  →  資料處理/  →  資料庫建置及產出/
（原始檔）    （轉MD+分類）    （調查計畫/工作底稿/分析報告）
```

## 先備條件

- Claude Code 桌面版（Pro 方案以上）
- Node.js
- Python 3.8+
- Obsidian（知識庫筆記軟體）
- 網路連線

## 設定流程（15 大步驟）

| 階段 | 步驟 | 說明 |
|------|------|------|
| 一 | 環境檢查 | 自動檢查 Node.js、Pandoc、Tesseract OCR、Python 套件、Selenium |
| 二 | 認識使用者 | 詢問職系、單位、語言偏好 |
| 三 | 安裝 Obsidian + 建立 Vault | 建立三層資料夾結構 |
| 四 | 連接 MCP | 安裝 mcpvault，讓 Claude Code 讀寫 Obsidian 筆記 |
| 五 | 建立 CLAUDE.md | 工作規則（分類規則、生成原則、批次轉換流程） |
| 六 | 建立 Templates | 工作日誌、週計畫、知識庫頁面模板 |
| 七 | 建立知識庫初始檔案 | index.md + log.md |
| 八 | 設定 Web Clipper | 將瀏覽器剪藏預設存至 `資料蒐集/` |
| 九 | 設定每週知識重整排程 | 每週日自動執行 Karpathy 式七步驟重整 |
| 十 | 建立歡迎筆記 + 驗證 | 15 項最終驗證清單 |
| 十一 | 生成調查計畫 | 調查計畫 + 法規連結 + 調閱清單 + 缺失問卷 |
| 十二 | 批次轉換既有檔案 | `/資料處理` 一鍵批次轉換 |

## 使用方式

安裝完成後：

| 你說的話 | 回應動作 |
|---------|---------|
| `/資料處理` | 批次轉換 `資料蒐集/` 中所有原始檔為 MD 並自動分類 |
| 「幫我生成一份[主題]調查計畫」 | 產出調查計畫 + 法規連結 + 調閱清單 + 缺失問卷（MD+WORD） |
| 「幫我根據[主題]生成工作底稿」 | 依四階段結構產出標準化底稿 |
| 「跑一次知識重整」 | 手動觸發完整七步驟知識重整 |
| 「搜尋筆記有沒有 XXX」 | 搜尋整個 vault |
